[담당 파일]
- src/patch.js 만 수정 가능
- 다른 파일은 수정하지 마세요

[담당 범위]
1) getNodeByPath(root, path)
- childNodes 인덱스로 순회
- path []면 root 반환

2) applyPatches(rootDom, patches)
- patch 순서대로 처리
- ADD / REMOVE / REPLACE / PROPS_UPDATE / TEXT_UPDATE 모두 반영
- ADD는 공통 규격대로 path를 "부모 경로", index를 삽입 위치로 해석
- REMOVE / REPLACE / TEXT_UPDATE / PROPS_UPDATE는 path를 대상 노드 경로로 해석

3) 내부 유틸
- applyPropsUpdate(node, setProps, removeProps)
- create node는 반드시 vdomToDom 사용(중복 구현 금지)

4) 루트 처리
- rootDom은 patch 대상 루트 노드로 가정
- path []에서 REMOVE / REPLACE가 들어올 수 있으므로 루트 교체 / 삭제 케이스를 방어적으로 처리
- 잘못된 path가 들어오면 조용히 무시하거나 경고 로그를 남기는 방식 중 하나로 일관되게 처리

[중요]
- 최소 DOM 조작 원칙 준수
- 위험한 태그(script) 처리 제한이 필요하면 안전 주석 추가
- diff / history / UI 로직은 건드리지 말 것
- vdomToDom은 팀원 A가 만든 함수를 사용
- 부분 업데이트가 눈에 보이도록 불필요한 전체 재생성 대신 patch 타입별 최소 DOM 조작을 우선할 것
- 발표 설명을 위해 어떤 DOM API(childNodes, replaceWith, remove, insertBefore, setAttribute 등)를 썼는지 정리 가능하게 구현할 것

[산출물]
- patch 타입별 DOM API 사용 요약
- 경로(path) 해석 규칙 명시
- 실패 가능 케이스(잘못된 path, 루트 교체 등) 방어 처리 여부 설명
