[담당 파일]
- src/vdom.js 만 수정 가능
- 다른 파일은 수정하지 마세요

[담당 범위]
1) domToVdom(node)
- Element / Text 노드 처리
- props는 attributes 순회로 객체화
- children은 childNodes를 순회하되, 공통 규격의 의미 있는 자식 노드만 children에 포함
- 공백-only 텍스트 노드 제거
- Comment 노드는 무시

2) vdomToDom(vnode)
- "#text" 처리
- element 처리 + props setAttribute
- children 재귀 append

3) cloneVdom(vdom)
- JSON 기반 깊은 복사 구현

4) 내부 보조 로직(필요 시)
- isMeaningfulNode(node) 또는 getMeaningfulChildNodes(node) 같은 내부 헬퍼 작성 가능

[중요]
- 공통 인터페이스 규격(함수명 / VDOM 포맷 / patch 포맷) 절대 변경 금지
- text node를 유지할 때는 text를 trim 해서 저장하지 말고 원본 text를 유지할 것
- input / textarea / select의 value / checked는 우선 attribute 기준으로만 처리하고, 주석으로 한계 명시
- diff / apply / history / UI 로직은 건드리지 말 것
- export / 전역 노출 방식은 기존 프로젝트 구조를 그대로 따를 것
- domToVdom의 children 순서가 이후 diff / patch path 기준이 되므로 의미 있는 노드 판별 규칙을 코드로 분명하게 유지할 것
