처음 보는 사람도 이해하는 가상 DOM 데모

웹페이지는 왜
바뀐 부분만 고칠까?

왼쪽은 지금 실제 화면이고, 오른쪽은 곧 바뀔 후보 화면입니다. 컴퓨터는 두 화면을 비교해서 달라진 부분만 찾아낸 뒤, 실제 화면에 꼭 필요한 만큼만 반영합니다.

가상 DOM 비교용 설계도

실제 화면을 바로 고치기 전에, JS 객체로 만든 가벼운 복사본입니다.

비교 달라진 부분 찾기

이전 설계도와 다음 설계도를 비교해 무엇이 바뀌었는지 정리합니다.

반영 실제 반영 목록

텍스트 변경, 속성 변경, 노드 추가/삭제처럼 필요한 작업만 모아둡니다.

이력 이전 화면 저장

반영이 끝난 상태를 저장해 두고 이전 상태 / 다음 상태로 다시 꺼내 볼 수 있습니다.

1단계

다음 화면을 먼저 만든다

입력창이나 프리셋 버튼으로 “어떤 화면이 되길 원하는지”를 먼저 정합니다.

2단계

컴퓨터가 차이를 찾는다

오른쪽 후보 화면을 보고, 제목이 바뀌었는지 목록이 추가됐는지 비교합니다.

3단계

실제 화면에는 필요한 만큼만 반영한다

반영 버튼을 누르면 바뀐 부분만 왼쪽 실제 화면에 적용되고 이력에도 저장됩니다.

지금 단계 준비 중
저장된 화면 0 / 0
바뀐 작업 수 0

발표용 빠른 시나리오

모르는 사람에게 보여줄 때는 한 번에 하나의 변화만 보여주는 것이 가장 쉽습니다.

프리셋은 현재 실제 화면을 기준으로 다음 초안을 자동으로 만듭니다.

발표 전에 시나리오 버튼을 눌러 “오른쪽 후보 화면”을 빠르게 준비할 수 있습니다.

1. 지금 실제 화면

메인 페이지 안에 들어 있는 독립 웹페이지입니다. 변화 반영이 성공할 때만 바뀝니다.

초록: 새로 추가 주황: 내용 또는 속성 변경 파랑: 태그 교체 빨강: 삭제가 일어난 위치

2. 다음으로 바뀔 후보 화면

메인 페이지 안에 들어 있는 또 다른 독립 웹페이지입니다. 실제 반영 전 비교용으로만 사용됩니다.

오른쪽 작은 웹페이지는 비교용 문서입니다. 공백만 있는 글자와 주석은 정규 규칙에 따라 제외됩니다.

3. 직접 수정해 보기

원하는 HTML을 적으면 오른쪽 후보 화면이 먼저 바뀌고, 변화 반영 버튼을 눌러야 왼쪽에 반영됩니다.

초기 DOM을 로드하는 중입니다.

맨 바깥 요소는 1개만 허용됩니다. 공백만 있는 글자와 주석은 자동으로 제외됩니다.

컴퓨터가 찾은 변화

개발 용어가 익숙하지 않아도 따라올 수 있도록 사람 말로 먼저 설명합니다.
